# PDF 清晰化工具 (pdf_clarity.py)

自动让扫描版 PDF 变清晰。**逐页分析内容类型**，对不同类型的页面用不同手段处理：

| 页面类型 | 判断依据 | 处理手段 |
|---|---|---|
| 原生文字页（本来就有可复制文字，不是扫描图） | 能提取到文字 且 页面不是整页大图 | 原样保留，不处理 |
| 文字扫描件（含独立彩色插图） | 图像统计特征（必要时用 OCR 二次确认）；页面内再检测彩色插图区域 | 插图区域：照片流程；其余文字区域：文字流程；两者羽化混合拼接 |
| 照片 / 图片扫描页 | 饱和度较高、色彩渐变多、非文字特征 | 彩色去噪 → LAB 空间 CLAHE 提升层次 → USM 锐化 → 轻微提升饱和度 |

**默认全程保留颜色**：即使一页被判定为"文字页"，也只在亮度通道上做增强，颜色通道不会被抹掉；只有显式加 `--binarize` 或 `--grayscale` 才会转黑白/灰阶。

**默认开启"同页混合处理"**：如果一页文字页里嵌了独立的彩色插图/照片（比如书名页里的配图），工具会自动把插图区域检测出来单独用照片流程处理（去噪、加层次、加饱和度），其余文字背景仍用文字流程处理，两者边界做羽化过渡后无缝拼接——插图和文字各自用最合适的手段，不用再"整页二选一"。判断插图区域时会过滤掉"看起来饱和度高但其实是彩色文字/标题"的情况（比如橙色大标题），避免把文字误当插图处理。

## 安装依赖

```bash
pip install pypdf pdfplumber pypdfium2 opencv-python-headless numpy pillow img2pdf pytesseract
```

如果需要 `--ocr`（给清晰化后的页面重新生成可搜索文字层），还需要系统装好 tesseract：

```bash
# Ubuntu/Debian
sudo apt-get install tesseract-ocr
# 如果要识别中文，还需要装语言包
sudo apt-get install tesseract-ocr-chi-sim
```

> 注意：有些系统 `python3` 指令没有正确指向装好依赖的解释器（比如多个 Python 版本共存、或 `python3` 没建立别名）。如果运行时"没有任何反应"，先试试把 `python3` 换成 `python`，并用 `python -c "import cv2"` 之类的命令确认依赖装在了你实际执行的那个解释器里。

## 用法

```bash
# 最简单：自动判断每页类型并增强，颜色默认保留
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf

# 页数多、追求速度：开多进程 + 加速去噪
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --fast --workers 8

# 提高渲染分辨率（文字很小、原图 DPI 很高时建议调大）
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --dpi 400

# 处理完顺便加上可搜索文字层（OCR）
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --ocr --lang eng
# 中文文档：--lang chi_sim（需已安装对应语言包）

# 强制全部按"文字页"处理（比如你确定整份文档都是文字，没有照片）
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --mode text

# 文字页做黑白二值化，锐利度最高，但会丢失颜色（适合纯黑白文档）
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --binarize

# 文字页转灰阶但不二值化（保留灰阶细节，但丢彩色）
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --grayscale

# 增强力度更强（噪点更多/更模糊的扫描件可以试试）
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --strength strong
```

## 参数一览

| 参数 | 说明 | 默认值 |
|---|---|---|
| `input` | 输入 PDF 路径（必填，位置参数） | - |
| `-o / --output` | 输出路径 | `原文件名_clear.pdf` |
| `--dpi` | 渲染分辨率 | 300 |
| `--mode` | `auto`/`text`/`photo` | auto |
| `--binarize` | 文字页黑白二值化（丢颜色） | 关闭 |
| `--grayscale` | 文字页转灰阶但不二值化（丢颜色） | 关闭 |
| `--strength` | `normal`/`strong` | normal |
| `--ocr` | 生成可搜索文字层 | 关闭 |
| `--lang` | OCR 语言（如 `eng`、`chi_sim`、`chi_sim+eng`） | eng |
| `--workers` | 并行处理的进程数 | 自动按 CPU 核心数 |
| `--fast` | 加速模式：分类跳过多余 OCR、去噪窗口缩小 | 关闭 |
| `--no-mixed` | 关闭同页混合处理（整页只用一种流程，旧行为） | 默认开启混合处理 |
| `--quiet` | 不打印进度 | 关闭 |

## 效果说明

- **文字扫描件**：去噪 + 自动纠偏（歪的扫描件会被转正）+ 局部对比度增强 + 锐化，文字边缘更清晰、背景更干净，且颜色默认保留。加 `--binarize` 可以做到打印稿一样的纯黑白效果，但如果原文档有颜色标记/高亮/插图，会丢失颜色信息。
- **照片/图片页**：彩色去噪 + 提升明暗层次 + 锐化 + 轻微加饱和度，让模糊、偏灰的照片看起来更清楚，同时保留自然色彩。
- **原生文字页**（PDF 本身自带文字层，不是图片扫描）：直接保留，不做任何处理，因为这类页面本身已经是矢量清晰的，处理反而会降低质量、增大文件体积。
- **同页混合内容**：如果文字页里嵌了独立的彩色插图，插图区域会被自动检测出来，单独用照片流程处理（保留自然色彩、加饱和度），文字背景则继续用文字流程（更锐利），两者用羽化蒙版无缝拼接，不会有明显接缝。可以用 `--no-mixed` 关闭这个行为，回到"整页统一处理"的旧逻辑。

## 处理速度

页数多时建议：
```bash
python pdf_clarity.py 扫描件.pdf -o 清晰版.pdf --fast --workers <你的CPU核心数>
```
- `--workers`：按页并行处理，多核机器上速度接近线性提升（默认已自动按核心数设置，不指定也会生效）
- `--fast`：分类阶段大幅减少 OCR 调用（大多数页面靠图像统计就能判断，只有模棱两可时才用 OCR 确认），并把去噪算法的搜索窗口调小，实测能带来数倍加速，画质差异通常不明显
- `--workers 1` 可以强制单进程顺序执行，方便调试或看逐页日志

## 已验证

- 用真实扫描书籍 PDF 测试过：文字页与照片页分类正确，文字清晰度（Laplacian 锐度指标）提升约 2 倍，对比度提升 40%+；彩色插图页颜色不再被误转灰阶。
- `--ocr` 生成的文字层经测试可以被正常提取（可复制/可搜索）。
- 多进程 (`--workers`) 与 `--fast` 模式均已验证输出页数、内容正确，`--fast` 相比默认模式实测提速约 3 倍。
- 同页混合处理已用真实书籍的书名页（大段标题文字 + 独立彩色插图）测试：插图区域被准确检测出来（排除了误将橙色标题文字当插图的情况），插图饱和度从原图的约 77 提升到约 82（增强生效），标题文字区域饱和度保持在约 18-20（未被误处理），边界处像素过渡平滑无接缝。

## 注意事项

- 分类是启发式判断，极端情况下（比如整页都是密集小字的照片，或者纯手写体）可能判断不准，可以用 `--mode text` 或 `--mode photo` 手动指定。
- 图像处理不能凭空"生成"扫描时丢失的细节，本质是去噪、增强对比度、锐化、纠偏，让原本模糊/偏暗/倾斜的扫描件看起来更清楚，而不是无中生有的超分辨率重建。
- 处理后的图片页会比原文件略大（因为提高了对比度/锐化，JPEG 压缩后细节更多）；如果需要更小文件体积可以调低 `--dpi`。
- 同页混合处理基于"彩色饱和度+填充率"的启发式判断插图区域，对纯手绘线稿插图（黑白线条、饱和度很低）可能识别不出来，此时会退回整页按文字流程处理；如果发现某页插图没被正确识别/处理效果不理想，可以用 `--mode photo` 强制整页按照片处理，或 `--no-mixed` 关闭这个功能。
- 开启 `--binarize` 或 `--grayscale` 时，同页混合处理会自动关闭（这两个参数本身就是要"整页统一效果"）。
